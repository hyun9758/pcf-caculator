# CarbonTrack - 쉽게 설명

---

## 이 프로젝트가 뭐야?

공장에서 제품 만들 때 **탄소(CO₂)가 얼마나 나오는지** 계산하고 보여주는 웹사이트.

예를 들어 철강 1톤 만들면:
- 철광석 캘 때 → 탄소 나옴
- 공장에서 녹일 때 → 탄소 나옴
- 트럭으로 배송할 때 → 탄소 나옴

이걸 전부 합치면 **"이 제품의 탄소 발자국"** = **PCF**.

---

## 만든 기능 6가지

### 1. 대시보드 (홈 화면)
회사 전체 탄소 배출 현황을 **한 눈에**.
- 총 배출량 숫자
- 어디서 많이 나오는지 파이 차트
- 분기별 줄고 있는지 추세 그래프
- 제품별 순위

**경영자 모드 / 실무자 모드** 토글:
- 경영자: 핵심 숫자만 (바쁘니까)
- 실무자: 상세 분석까지 (업무에 필요하니까)

### 2. 제품 상세 (Sankey 다이어그램)
제품 하나 클릭하면 **탄소가 어디서 얼마나 나오는지 흐름도**로 보여줌.

```
철광석 ──────▶ 원소재 취득 ──────▶
석탄 ─────────▶                    ──▶ 총 1,855 kgCO₂e
천연가스 ────▶ 제조 ──────────────▶
전력 ─────────▶
트럭 ────────▶ 운송 ──────────────▶
```
선이 굵을수록 = 탄소 많이 나옴. 비전문가도 직관적으로 이해 가능.

### 3. 데이터 입력
"우리 제품에 철광석 1,600kg 들어가" → 입력하면 즉시 CO₂e 계산.
- 추가/삭제 실제 동작
- 오른쪽에 실시간으로 PCF 변화 확인

### 4. 제품 비교
"열연강판 vs 노트북 vs PET병" 나란히 비교.
- 어떤 제품이 탄소를 많이 배출하는지
- 어떤 단계에서 차이나는지

### 5. 감축 시뮬레이션
**"만약에?"** 시나리오.
- "전기를 태양광으로 바꾸면?" → -30% 감축
- "트럭 대신 철도 쓰면?" → -15% 감축
- 여러 개 동시에 적용 → 누적 효과 확인

### 6. 반응형
핸드폰에서도 사용 가능. 메뉴 접히고, 로딩 중엔 스켈레톤 표시.

## 기술적으로 뭘 한 거야?

| 뭘 했나 | 왜 했나 |
|---------|---------|
| **Next.js** (웹 프레임워크) | 과제 요구사항 |
| **TypeScript** (타입 안전) | "원소재"에 오타 써도 컴파일 에러로 잡힘 |
| **Nivo** (차트 라이브러리) | Sankey 다이어그램 지원하는 유일한 React 차트 |
| **PostgreSQL** (데이터베이스) | 데이터 영구 저장. Neon 클라우드 DB 사용 |
| **Repository 패턴** | DB 없어도 동작, DB 있어도 동작. 환경변수 하나로 전환 |
| **REST API** (10개) | 프론트/백 분리. 나중에 모바일 앱도 같은 API 사용 가능 |

---

## 데이터는 어디서?

실제 공개 자료 기반 Mock 데이터:
- **ecoinvent** — 세계 최대 환경 데이터베이스
- **IPCC** — UN 기후변화 패널
- **환경부** — 한국 전력 배출계수 (0.459 kgCO₂e/kWh)

5개 제품 각각 다른 탄소 특성:

| 제품 | 탄소 많이 나오는 곳 |
|------|-------------------|
| 열연강판 | 공장 (석탄 태우는 고로) |
| 노트북 | 부품 (반도체, 희금속 채굴) |
| 배터리셀 | 제조 (전기 많이 씀) |
| 골판지 박스 | 원료 (펄프) |
| PET병 | 원료 (석유) |

## 파일 구조 (간단 버전)

```
src/
├── app/          → 페이지들 (대시보드, 제품, 입력, 비교, 시뮬레이션)
├── components/   → 재사용 가능한 UI 조각들 (차트, 카드, 테이블)
├── lib/
│   ├── types/    → "제품이란 이런 모양" 타입 정의
│   ├── calculations/ → CO₂e 계산 공식
│   └── data/     → 데이터 + DB 연결
└── contexts/     → 경영자/실무자 모드 상태 관리
```
